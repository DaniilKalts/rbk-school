# Неделя 3 - Weather + Users API

## Тема

Разработка REST API сервиса на Go с использованием PostgreSQL для управления пользователями и их городами, интеграция с внешним API погоды, и работа с историей запросов.

## Проект

Реализовать REST API сервис на Go с PostgreSQL, который:
- управляет пользователями
- позволяет пользователю следить за несколькими городами
- получает погоду из внешнего API
- сохраняет историю погодных запросов
- поддерживает **фильтрацию истории по городу**

## Требования

### REST API

#### Пользователи
- `POST /users`
- `GET /users`
- `GET /users/{id}`
- `PUT /users/{id}`
- `DELETE /users/{id}` (soft delete)

#### Города пользователя
- `POST /users/{id}/cities` — добавить город
- `GET /users/{id}/cities` — список городов пользователя
- `DELETE /users/{id}/cities/{city_id}` — удалить город

#### Погода
**Получить погоду по всем городам пользователя:**

`GET /users/{id}/weather`

👉 **Должен:**
- сходить во внешний API
- вернуть агрегированный ответ
- сохранить данные в историю

**⚙️ Важная бизнес-логика:**
При запросе `GET /users/{id}/weather` сервис должен:
1. Получить пользователя
2. Проверить `deleted_at IS NULL`
3. Получить список городов
4. Получить погоду по каждому городу
5. Сохранить каждую запись в `weather_history`

### История погоды (ОСНОВНОЕ ТРЕБОВАНИЕ)

**🔥 Получить историю с фильтрацией**

`GET /users/{id}/weather/history?city=Almaty`

📌 **Требования:**
- параметр `city` — **обязательный**
- возвращает только записи по указанному городу
- сортировка: `ORDER BY requested_at DESC`
- поддержка лимита: `GET /users/{id}/weather/history?city=Almaty&limit=10`

📦 **Пример ответа:**
```json
{
  "user_id": 1,
  "city": "Almaty",
  "history": [
    {
      "temperature": 18,
      "description": "cloudy",
      "requested_at": "2026-04-20T10:00:00Z"
    },
    {
      "temperature": 20,
      "description": "sunny",
      "requested_at": "2026-04-19T10:00:00Z"
    }
  ]
}
```

### База данных

🧱 **Структура БД (дополнение)**

Таблица `weather_history`:
- `id`
- `user_id`
- `city`
- `temperature`
- `description`
- `requested_at`

👉 **Важно:** индекс по `(user_id, city)`

⚠️ **Важные требования**

**1. Без SQL injection**

Параметр `city` приходит от пользователя!

❌ Нельзя:
```sql
"SELECT * FROM weather_history WHERE city = '" + city + "'"
```

**2. Производительность**
- обязательно использовать индекс `(user_id, city)`
- не делать лишние запросы (N+1)

**3. Архитектура**
- `handler → service → repository`
- внешний Weather API → отдельный `client`
